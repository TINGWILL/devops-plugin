# 固定列背景透明问题原因分析

## 问题现象
当鼠标悬停在未选中行时，选中行的固定列（复选框列、固定左侧列、固定右侧列）背景变透明。

## 根本原因

### 1. CSS 选择器优先级冲突
- **固定列 hover 样式**（第109-114行）：优先级不够高
- **固定列选中行样式**（第116-121行）：虽然使用了 `!important`，但可能被 Semi Design 内部样式覆盖

### 2. 选中行通用样式影响
- **选中行样式**（第250-252行）：`tr.semi-table-row-selected td` 会影响所有 td，包括固定列
- 这个规则可能覆盖了固定列的专门样式

### 3. CSS 特异性问题
选中行固定列的样式特异性可能不够高，无法覆盖：
- Semi Design 内部样式
- 其他 hover 状态的样式
- 通用选中行样式

## 解决方案

### 方案1：提高选中行固定列样式特异性（推荐）
在选中行固定列样式中同时使用 `:hover` 和 `:not(:hover)` 伪类，确保无论鼠标在哪里，选中行的固定列都有背景色。

### 方案2：排除固定列不受通用样式影响
修改通用选中行样式，使用 `:not()` 排除固定列，让固定列使用自己的专门样式。

### 方案3：使用更高优先级的选择器
组合使用多个类选择器和属性选择器，提高样式优先级。

### 方案4：JavaScript 动态设置（最后手段）
如果 CSS 无法解决，使用 JavaScript 在运行时强制设置固定列背景色。

## 最佳实践
优先使用方案1和方案2的组合：
1. 排除固定列不受通用样式影响
2. 提高选中行固定列样式的优先级和特异性
